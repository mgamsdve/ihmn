# Analyse de l'expérience mobile — IHMN

## Description

Cette analyse porte sur le comportement du site IHMN sur mobile.

L'objectif est d'évaluer :

- la navigation mobile
- la lisibilité
- la navigation dans les contenus
- les problèmes d'ergonomie
- les problèmes techniques liés à l'affichage.

Les observations ont été réalisées en consultant plusieurs pages du site sur un écran mobile.

---

# 1. Navigation mobile

Sur mobile, la navigation principale devient un **menu hamburger** situé dans le header.

Lorsque l'utilisateur clique sur l'icône :

- un panneau latéral s'ouvre
- le menu apparaît sous forme de liste verticale.

Structure du menu :

- Accueil
- Formations
- IHMN
- FAQ
- Thérapeutes
- Contact

Les sous-pages sont accessibles via un système d'accordéon.

---

# 2. Problème de lisibilité du menu

Dans le menu mobile, les sous-items de "Formations" apparaissent avec un **contraste extrêmement faible**.

Le texte est affiché en gris clair sur fond clair.

Conséquence :

- les sous-pages deviennent difficilement lisibles
- l'utilisateur peut penser que les liens sont désactivés.

Ce problème constitue un **défaut d'accessibilité**.

---

# 3. Problème d'expérience utilisateur

L'ouverture du menu mobile masque la page, mais :

- la structure du menu reste peu lisible
- l'utilisateur doit scroller pour accéder à toutes les entrées.

La hiérarchie n'est pas clairement visible.

---

# 4. Problème de navigation dans les contenus

Sur plusieurs pages, les contenus sont très longs et nécessitent un **scroll important**.

Exemple :

- pages de modules de formation
- pages descriptives.

Les modules sont affichés sous forme de longues listes.

Conséquences :

- navigation difficile
- perte de repères pour l'utilisateur.

---

# 5. Problème d'ergonomie : modules de formation

Les modules apparaissent sous forme de blocs contenant :

- un titre
- un extrait
- un bouton "description".

Cependant :

- l'utilisateur doit scroller longuement
- l'organisation par année n'est pas toujours claire.

La navigation dans le programme devient donc peu intuitive.

---

# 6. Problème d'affichage des widgets

Sur certaines pages, des widgets externes apparaissent mal adaptés au mobile.

Exemple :

widget Facebook.

Le widget affiche :

- des blocs vides
- une grille mal adaptée à la largeur de l'écran.

Conséquence :

- perte d'espace
- dégradation de l'expérience utilisateur.

---

# 7. Bouton "retour en haut"

Un bouton flottant "retour en haut" est affiché en permanence.

Problème :

- il peut masquer certains éléments
- il occupe une partie de l'écran.

---

# 8. Footer mobile

Le footer est affiché sous forme de blocs empilés :

- lieu des cours
- siège social
- contact.

Observation :

La lisibilité reste correcte.

Cependant :

- les sections sont très espacées
- la navigation reste limitée.

---

# 9. Performance et perception mobile

Le site semble avoir été conçu principalement pour desktop.

Sur mobile :

- certains blocs prennent trop de place
- la navigation nécessite beaucoup de scroll
- certains éléments ne sont pas optimisés.

---

# 10. Analyse UX

Points positifs :

- header simple
- navigation identifiable
- contenu accessible.

Points problématiques :

- lisibilité insuffisante dans certains menus
- navigation longue dans les contenus
- widgets externes mal intégrés
- structure pédagogique difficile à parcourir.

---

# 11. Recommandations pour la refonte

Pour améliorer l'expérience mobile :

### améliorer le contraste du menu

Les sous-menus doivent être clairement lisibles.

---

### restructurer les programmes de formation

Créer des pages distinctes pour :

- chaque année
- chaque module.

---

### supprimer les widgets inutiles

Les widgets externes doivent être adaptés au mobile ou supprimés.

---

### limiter le scroll

Réorganiser les contenus afin d'éviter des pages trop longues.

---

# Conclusion

L'expérience mobile actuelle du site IHMN présente plusieurs limites :

- problèmes de lisibilité
- navigation longue
- structure peu adaptée aux petits écrans.

Une refonte du site permettra d'améliorer :

- la navigation
- l'accessibilité
- l'expérience utilisateur mobile.
